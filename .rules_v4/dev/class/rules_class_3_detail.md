## クラス詳細設計書
  クラスの詳細設計書になります。


### 概要
  そのクラスがどんな処理を行うのかを記述します。


#### クラス全体の依存関係

  クラス全体として保持・参照する依存関係をテーブル形式で記述します。
  下記を記述します。

  - 依存先クラス名
  - 依存種別
  - 使用元メソッド
  - 使用先メソッド / フィールド
  - 目的
  - 備考

  依存種別は下記とします。
  - 呼び出し
  - 継承
  - 実装
  - 注入
  - 参照


### フィールド

  定義しているフィールド一覧をテーブル形式で記述します。
  下記を記述します。
  - フィールド名
  - 可視性
  - 属性
  - データ型
  - 説明

  可視性は下記とします。
  - private
  - public
  - protected
  - package-private / internal

  属性は下記とします。
  - const
  - static
  - readonly
  - なし


### メソッド

  定義しているメソッド一覧を記述します。
  下記を記述します。
  - メソッド名
  - 可視性
  - 引数一覧をテーブル形式で記述します。
  - 戻り値を記述します。
  - 依存関係
  - 処理内容
  - 例外処理
  - 永続化
  - トランザクション
  - ログ / 監査

  内部変数は記述する必要はありません。

  可視性は下記とします。
  - public
  - private
  - protected
  - package-private / internal

  戻り値は原則として1つとし、下記を記述します。
  - データ型
  - 名称
  - 説明


#### メソッド単位の依存関係

  各メソッドが実際に利用する依存関係をテーブル形式で記述します。
  下記を記述します。

  - 依存先クラス名
  - 依存種別
  - 使用元メソッド
  - 使用先メソッド / フィールド
  - 目的
  - 備考

  依存種別は下記とします。
  - 呼び出し
  - 継承
  - 実装
  - 注入
  - 参照
  

#### 例外処理

  各メソッドについて、必要な例外処理をテーブル形式で記述します。
  下記を記述します。

  - 例外名
  - 発生メソッド
  - 発生条件
  - 呼び出し元への影響
  - 外部返却内容
  - ログ出力有無
  - 備考


#### 永続化

  各メソッドについて、永続化処理がある場合はテーブル形式で記述します。
  下記を記述します。

  - 対象メソッド
  - 種別
  - 対象テーブル / エンティティ
  - 条件
  - 備考


#### トランザクション

  各メソッドについて、トランザクション制御がある場合はテーブル形式で記述します。
  下記を記述します。

  - 対象メソッド
  - 開始有無
  - 対象処理
  - コミット条件
  - ロールバック条件
  - 備考


#### ログ / 監査

  各メソッドについて、主要なログまたは監査処理がある場合はテーブル形式で記述します。
  下記を記述します。

  - 対象メソッド
  - ログ種別
  - 出力条件
  - 主な出力項目
  - 監査対象有無
  - 備考


### 禁止事項

  - 公開メソッドから他の公開メソッドを呼び出すことは原則非推奨とする。
  - 呼び出す場合は、責務分離上の理由と再利用上の妥当性を記述すること。
  - 単なる処理流用が目的であれば、非公開メソッドへ切り出すことを優先すること。


### 特記事項

  その他、特記事項がある場合はこちらに記述します。
