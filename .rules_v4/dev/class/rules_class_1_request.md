## 要件定義書

### 目的

クラス設計を開始する前に、クラスの責務、利用場面、公開インターフェース、依存関係、受入条件を整理します。
この手順で作成した要件定義書を、以降の基本設計書、詳細設計書、実装の入力として扱います。

### 要件定義書の内容

下記を記述します。

|no|項目|説明|
|---|---|---|
|1|目的・背景|なぜそのクラスが必要なのか、何を解決するのかを記述します。|
|2|責務|クラスが担う責務と担わない責務を記述します。|
|3|利用場面|どこから、どのように呼び出されるかを記述します。|
|4|入出力|コンストラクタ、メソッド、戻り値、例外、状態変更を記述します。|
|5|依存関係|参照するクラス、外部API、設定、データ構造を記述します。|
|6|制約条件|命名規則、既存仕様、継承やインターフェースの制約を記述します。|
|7|受入条件|完了判定に使う確認項目をチェックリスト形式で記述します。|
|8|未決事項|確認が必要な点、保留している判断、ユーザ確認が必要な点を記述します。|

### 作成手順

1. ユーザの依頼内容を分解し、曖昧な点を洗い出します。
2. 確認済みの内容、推測している内容、未確認の内容を分けて記述します。
3. 既存仕様を変更する場合は、[ADD] [MOD] [DEL] [KEEP] のタグを使って変更種別を明示します。
4. 未決事項がある場合は、基本設計へ進む前にユーザへ確認します。
5. 出力ファイル名は `{filename}_v{n}.{m}_1_request.md` とします。
