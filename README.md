# product_agent_rules

AI エージェント向けの開発・確認ルールを管理するためのリポジトリです。

複数バージョンの `.rules_*` ディレクトリを保持し、別プロジェクトでは必要なルールセットを `.rules` として取り込んで利用します。

## 使用準備

### 手順1. 取り込み先プロジェクトへルールを配置する

別プロジェクトへ取り込む場合は、`pull_rules.sh` を実行してください。

最新版の `.rules_v5` が、取り込み先リポジトリでは `.rules` という名前で配置されます。

### 手順2. AI エージェント用の指示を追加する

取り込み先プロジェクトの `AGENTS.md` など、AI エージェントが参照する指示ファイルには、以下を追加してください。

```markdown
- 開発ルール
  - 開発を行う場合、必ず`.rules/dev/rules_dev.md`を確認すること。
```

## 使用方法

### 簡単な使用例

開発を依頼する前に、使用する開発ルールの種類を選びます。v5 の開発ルールは、用途ごとに `app`、`batch`、`class` に分かれています。
v5 では、開発手順の最初に要件定義書を作成してから、基本設計書以降へ進みます。

- `app`: アプリケーション開発向けのルールです。画面、機能、構成、実装、確認までの流れを扱います。
- `batch`: バッチ処理やスクリプト処理向けのルールです。定期実行、データ処理、自動処理などを作る場合に使用します。
- `class`: クラス設計向けのルールです。現在は作り下であるため、利用時は内容を確認してください。

依頼例:

```text
.rules/dev/rules_dev.md を確認してから進めてください。
今回は app ルールを使用します。
対象は sample_app です。
まず要件定義書を作成してください。
```

## ディレクトリ構成

```text
.rules_v1/
.rules_v2/
.rules_v3/
.rules_v4/
.rules_v5/
pull_rules.sh
```

- `.rules_v1`: 初期版のルールセット
- `.rules_v2`: v2 のルールセット
- `.rules_v3`: v3 のルールセット
- `.rules_v4`: v4 のルールセット
- `.rules_v5`: 最新版のルールセット
- `pull_rules.sh`: 他リポジトリから最新版の `.rules_v5` を取り込むスクリプト

## ルールの取り込み

取り込み先のリポジトリで `pull_rules.sh` を実行します。

```bash
bash ./pull_rules.sh
```

既定では `yuki746289/product_agent_rules` から取得します。

### 取り込み対象

スクリプトを実行すると、取得元リポジトリにある `.rules_v5` が `.rules` として配置されます。

```text
.rules_v5 -> .rules
```

### 上書き確認

取り込み先に既存の `.rules` ディレクトリがある場合は、置換前に確認メッセージが表示されます。

```text
Replace '.rules'? [y/yes/n/no]:
```

`y` または `yes` を入力すると、既存の `.rules` をフォルダごと削除してから、`.rules_v5` を `.rules` としてコピーします。

`n` または `no` を入力すると、処理を中止します。

### 注意

- `.rules` はフォルダごと置換されます。
- ルールセットごとにファイル名や構成が異なる場合でも、古いファイルが残らないようにしています。
- 取り込み後は `git status --short` や `git diff --stat` で差分を確認してください。
